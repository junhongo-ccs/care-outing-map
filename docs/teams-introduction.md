# Teams 紹介文

東京都の福祉トイレ公開データを使って、外出前・外出中に使える「東京おでかけ安心3Dマップ」を試作しました。

同じローカルフォルダをcodexとclaudeでお互いを補完させたりレビューさせて作成・調整しています。

公開URL:
https://care-outing-map.onrender.com/

きっかけのひとつは、車椅子を使っている義母が、最近義理の妹と美術館へ出かけたことです。外出前に「このあたりなら、どのトイレが使いやすそうか」をPCで計画できること、また外出先で急に探したくなったときにスマホでも使えることを意識しています。

主な特徴:

- 東京都内の福祉トイレ 8,944件を収録
- Google Maps JavaScript API の3D地図機能を利用
- PCでは事前計画、スマホでは外出先での緊急検索を想定
- 詳細カードから、トイレ入口写真だけでなく内部設備写真も確認可能
- 写真はアプリ側に画像ファイルを保持せず、データ内の画像URLを参照
- 指定エリア内のトイレだけを表示し、東京中の何千件ものピンを一度に描画しない構成
- ピンの色は、地図に埋もれにくく識別しやすい色に調整
- 選択中ピンには独自SVGグリフを表示

ケアモード:

- 車椅子の家族
- 乳幼児連れ
- 高齢者同伴

スコアは、ケアモードごとに重視する設備を変えています。
車椅子モードでは、車椅子出入り可否・転回スペース・手すりを重視しています。
乳幼児連れモードでは、おむつ台・ベビーチェアの有無を重視しています。
高齢者同伴モードでは、手すりの有無を最重視し、出入りのしやすさも加味しています。

なお、全モード共通で、検索地点からの距離や写真の有無もスコアに少し反映しています。細かい配点はREADMEに記載しています。

●実装メモ:

- フロントエンドは React + TypeScript + Vite
- 地図は Google Maps JavaScript API の `maps3d` ライブラリを利用
- `Map3DElement` で3D地図を生成
- `MarkerInteractiveElement` と `PinElement` で福祉トイレのピンを表示
- 選択中ピンは `glyphSrc` で独自SVGを差し込み
- データはGeoJSONとして前処理済み
- APIキー未設定や3D地図の初期化失敗時は、静的プレビュー地図にフォールバック

●データ出典とライセンス:

- データ名: 車椅子使用者対応トイレのバリアフリー情報
- 公開元: 東京都
- 掲載ページ: https://catalog.data.metro.tokyo.lg.jp/dataset/t000054d0000000342
- ライセンス: クリエイティブ・コモンズ 表示 4.0 国際（CC BY 4.0）
- 加工内容: 検索・表示用にGeoJSONへ変換し、エリア中心からの距離、ケアモード別スコア、表示用IDなどを付与

●少し苦労したところ:

Google 3D Mapsは、通常の地図表示と同じ感覚ではそのまま出ませんでした。`maps3d` / `Map3DElement` を使うには、Google Maps JavaScript API を `v=beta` で読み込む必要がありました。

そのため、専用の `googleMapsLoader.ts` を用意し、API scriptを動的にロードして、読み込み完了後に `window.google.maps.importLibrary("maps3d")` を呼べるようにしています。このローダーがない状態では3D地図がうまく表示されず、ここは実装上のひとつの山でした。

また、東京都全体では8,944件のトイレデータがありますが、全部を一度に地図へ出すと重くなります。そこで「お出かけ先の周辺を見る」という用途に割り切り、検索地点や選択エリアの周辺だけを描画するようにしました。Google 3D Map側は、東京全域の網羅表示ではなく、外出先を立体的に把握するためのビューワーとして使っています。

詳細カードの写真も、何千件分の画像ファイルをアプリ内に保持しているわけではありません。データ内に画像URLが入っていたため、それを参照して入口写真や内部設備写真を表示しています。このおかげで、アプリ自体は比較的軽く保てています。

美術館、駅、公民館、商業施設など、場所ごとにトイレ設備や写真を見比べるだけでもかなり発見があります。高齢者・車椅子利用だけでなく、乳幼児連れのおむつ台探しにも使えるので、ぜひ触ってみてください。
